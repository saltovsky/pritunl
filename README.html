📋 Описание

Данный проект предоставляет готовую конфигурацию для запуска Pritunl VPN сервера в Docker-контейнере. Решение включает в себя:

🔷 Pritunl

Сервер для управления OpenVPN подключениями с веб-интерфейсом

🗄️ MongoDB

База данных для хранения конфигурации, пользователей и сессий

⚡ Автоматизация

Скрипт инициализации настраивает параметры при первом запуске

🚀 Быстрый старт

1. Клонирование и подготовка

# Создайте директорию для проекта
mkdir -p ~/pritunl-docker && cd ~/pritunl-docker

# Скопируйте файлы проекта (Dockerfile, compose.yml, init.sh, rootfs/)
# или клонируйте репозиторий

2. Запуск

# Сборка и запуск контейнеров
docker compose up -d --build

# Проверка статуса
docker compose ps

# Просмотр логов
docker compose logs -f pritunl

3. Первоначальная настройка

# Получить ключ установки (выполнить после старта контейнера)
docker exec pritunl pritunl default-key

# Или посмотреть в логах
docker compose logs pritunl | grep "default key"
📝 Далее:
  1. Откройте в браузере: https://<IP-адрес-сервера>
  2. Введите полученный ключ установки
  3. Создайте учетную запись администратора
  4. Настройте организацию и пользователей

⚙️ Конфигурация

Переменные окружения

Переменная Описание По умолчанию
TZ Часовой пояс Europe/Moscow
SERVER_PORT Порт веб-интерфейса 443
ACME_DOMAIN Домен для Let's Encrypt SSL —
MONGODB_SERVER Имя хоста MongoDB mongo

Пример с ACME (Let's Encrypt)

environment:
  - TZ=Europe/Moscow
  - SERVER_PORT=443
  - ACME_DOMAIN=vpn.example.com
⚠️ Важно: Для работы ACME домен должен указывать на ваш сервер, порт 80 должен быть открыт.

📁 Тома (Volumes)

Путь на хосте Путь в контейнере Описание
./db /data/db Данные MongoDB
./pritunl /var/lib/pritunl Конфигурация Pritunl, сертификаты
/etc/localtime /etc/localtime Синхронизация времени (read-only)

Создание директорий

mkdir -p db pritunl
chmod 755 db pritunl

🔌 Порты

Порт Протокол Описание
80 TCP HTTP (ACME challenge / редирект)
443 TCP HTTPS веб-интерфейс
1194 TCP/UDP OpenVPN подключения
1195 UDP Альтернативный OpenVPN порт

🛠️ Управление

# Перезапуск сервисов
docker compose restart

# Полная пересборка
docker compose up -d --build --force-recreate

# Остановка
docker compose down

# Остановка с удалением томов (⚠️ данные будут потеряны!)
docker compose down -v

# Вход в контейнер для отладки
docker exec -it pritunl /bin/bash

# Доступ к MongoDB
docker exec -it pritunldb mongosh

🔧 Troubleshooting

Контейнер не запускается

# Проверить логи
docker compose logs pritunl

# Проверить доступность MongoDB
docker exec pritunl nc -zv mongo 27017

VPN не подключается

# Проверить, включена ли переадресация на хосте
sysctl net.ipv4.ip_forward

# Включить при необходимости
echo 1 > /proc/sys/net/ipv4/ip_forward

# Проверить iptables на хосте
iptables -L -n -v

Проблемы с SSL/ACME

# Убедитесь, что порт 80 открыт
# Проверьте, что домен резолвится на ваш IP
dig vpn.example.com

# Пересоздайте контейнер после настройки домена
docker compose up -d --force-recreate

Сброс пароля администратора

docker exec pritunl pritunl reset-password

🔒 Безопасность

Рекомендация Статус
Использовать фиксированную версию MongoDB ✅
Не использовать privileged без необходимости ⚠️
Ограничить доступ к портам фаерволом 🔶
Регулярно обновлять образы 🔶
Использовать ACME для SSL ✅

Фаервол (пример для UFW)

ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 1194/tcp
ufw allow 1194/udp
ufw enable

📦 Обновление

# Остановить текущие контейнеры
docker compose down

# Пересобрать образ (если меняли Dockerfile)
docker compose build --no-cache

# Запустить заново
docker compose up -d

# Проверить версию в веб-интерфейсе
